This guide walks you through building and rendering HTML components with Aether from scratch. By the end you will have a working component, understand how dynamic content works, and see how to wire up HTMX attributes.

<Steps>
  <Step title="Install Aether">
    Install `pyaether` from PyPI:

    ```bash
    pip install pyaether
    ```

    Aether requires Python 3.10 or later. See [Install Aether](/installation) for full requirements and dependency pinning guidance.
  </Step>

  <Step title="Import elements">
    Aether organises HTML elements as Python classes under `aether.tags.html`. Import the elements you need alongside the `render` function:

    ```python
    from aether.tags.html import H1, Div, P
    from aether import render
    ```

    Every standard HTML5 element is available. The full list is in the [HTML tags reference](/reference/html-tags).
  </Step>

  <Step title="Build a component">
    Construct a component by instantiating an element with its attributes, then calling it with its children:

    ```python
    page = Div(_class="container")(
        H1()("My Awesome Page"),
        P()("This is a paragraph of text.")
    )
    ```

    The two-step syntax — `Element(**attributes)(children)` — separates attribute declaration from child composition. Attributes are keyword arguments; children are positional arguments in the call.

    **Attribute naming:** Python does not allow hyphens in keyword argument names, so Aether converts them automatically:
    - Leading underscore is stripped: `_class` → `class`, `_id` → `id`
    - Underscores become hyphens: `hx_get` → `hx-get`, `data_value` → `data-value`
  </Step>

  <Step title="Render to HTML">
    Pass your component to `render()` to get an HTML string:

    ```python
    print(render(page))
    ```

    Output:

    ```html
    <div class="container"><h1>My Awesome Page</h1><p>This is a paragraph of text.</p></div>
    ```

    All string content is HTML-escaped automatically. See [Safe strings](/concepts/safe-strings) for details on the escaping behaviour and `mark_safe`.
  </Step>
</Steps>

## Dynamic content

Because components are plain Python, you can use loops, conditionals, and comprehensions to build dynamic content. Generator expressions are supported directly in the call syntax:

```python
from aether.tags.html import Div, Li, Span, Ul
from aether import render

items = ["apple", "banana", "cherry"]
item_list = Ul()(Li()(item) for item in items)

name = "Alice"
greeting = Span()(f"Hello, {name}!")

print(render(Div()(greeting, item_list)))
```

Output:

```html
<div><span>Hello, Alice!</span><ul><li>apple</li><li>banana</li><li>cherry</li></ul></div>
```

You can pass any iterable — lists, generator expressions, or custom iterators — as children.

## HTMX integration

Aether's attribute naming convention makes HTMX attributes straightforward. Any `hx_*` keyword argument becomes the corresponding `hx-*` HTML attribute:

```python
from aether.tags.html import Button, Div, Img
from aether import render

click_to_load = Div(id="replace_me")(
    Button(
        _class="btn primary",
        hx_get="/example/?data=1",
        hx_target="#replace_me",
        hx_swap="outerHTML",
    )(
        "Load More Data...",
        Img(_class="htmx-indicator", src="/img/bars.svg", alt="Loading..."),
    )
)

print(render(click_to_load))
```

Output:

```html
<div id="replace_me"><button class="btn primary" hx-get="/example/?data=1" hx-target="#replace_me" hx-swap="outerHTML" type="button">Load More Data...<img class="htmx-indicator" src="/img/bars.svg" alt="Loading..." /></button></div>
```

The same convention works for Alpine.js (`x_data` → `x-data`), `data_*` attributes, and any other hyphenated attribute. See [HTMX](/plugins/htmx) and [Alpine.js](/plugins/alpinejs) for full integration guides.
